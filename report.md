# Institutional China ETF Trend Following Portfolio v3.0

## 一、策略定位

本策略是一套面向中国A股市场的机构级ETF趋势跟踪系统（Institutional China ETF Trend Following Portfolio）。

系统不预测市场顶部与底部，不试图抄底或逃顶，而是通过趋势识别、横截面动量比较以及风险控制机制，持续配置于市场中最强势的ETF资产。

策略完全基于量化规则运行，不依赖宏观预测、基本面判断或主观择时。

策略属于：

* Long Only CTA
* ETF Rotation
* Trend Following
* Cross-Sectional Momentum
* Risk Managed Portfolio

适用于：

* 普通A股账户
* PTrade自动化交易
* 10万至100万资金规模
* 日线级别交易
* 中长期资产配置

---

# 二、核心设计原则

系统遵循以下原则：

1. 只做多，不做空
2. 只交易ETF，不交易个股
3. 不使用融资融券
4. 不使用杠杆
5. 不预测市场
6. 所有决策基于收盘数据
7. 所有交易于下一交易日执行
8. 优先控制回撤而非追求收益最大化

系统通过：

市场过滤 → 动量排序 → 趋势质量过滤 → 流动性过滤 → 风险控制

五层结构完成投资决策。

# 六、趋势质量评分

高收益率并不一定代表高质量趋势。

系统引入趋势质量评分以识别：

* 趋势是否稳定
* 趋势是否连续
* 趋势是否具备可持续性

---

## 回归周期

120个交易日

---

## 线性回归斜率

计算：

Slope

衡量价格上涨速度。

---

## 趋势稳定度

计算：

R²

衡量价格走势与趋势线的拟合程度。

---

## 趋势质量得分

Quality Score

=

Standardized(Slope × R²)

该方法能够同时惩罚：

* 斜率不足
* 趋势震荡
* 非持续性上涨

相比简单加权方法更加稳健。

# PTrade 实施与系统架构设计

## 一、实施原则

本策略运行于国金证券 PTrade 交易平台。

由于 PTrade 当前运行机制存在以下限制：

* 策略只能提交单个 Python 脚本运行
* 不支持自定义 Python Package
* 不支持项目目录结构
* 不支持跨文件 import 自定义模块
* 回测与实盘运行环境保持一致

因此本策略采用：

**单脚本架构（Single File Architecture）**

而非传统量化系统常见的：

```text
strategy/
research/
broker/
data/
config/
```

多目录工程结构。

所有逻辑统一维护于一个脚本文件中。

---

## 二、回测框架决策

### 最终结论

采用：

**PTrade原生回测系统**

不开发独立回测引擎。

### 原因

对于本策略：

* 资金规模：10万元
* 交易标的：A股ETF
* 数据频率：日线
* 调仓频率：周度/月度
* 策略类型：CTA趋势跟踪

PTrade原生回测已经满足：

* 历史K线获取
* 调仓模拟
* 手续费模拟
* 净值统计
* 回测报告生成

需求。

自行开发回测框架需要额外处理：

* 复权问题
* ETF分红
* 停牌
* 涨跌停
* 滑点模型
* 成交量限制
* 组合净值计算

开发成本高于收益。

因此：

### 系统定位

PTrade负责：

* 数据获取
* 回测执行
* 实盘交易

策略代码负责：

* 信号计算
* 排名计算
* 风险控制
* 仓位管理

---

# 八、最终架构原则

本项目采用：

**单脚本运行 + 逻辑分层设计 + PTrade原生回测**

架构原则如下：

1. 不开发独立回测引擎。
2. 使用PTrade原生回测。
3. 所有策略逻辑与PTrade API解耦。
4. 所有PTrade接口统一映射。
5. 所有逻辑保存在单个脚本文件。
6. 代码按逻辑分层而非文件分层组织。
7. 回测与实盘共用同一套代码。
8. 优先保证稳定性与可维护性，而非工程复杂度。
9. 策略开发过程中，所有新增功能必须优先考虑 PTrade 兼容性，而不是研究环境兼容性。

该方案最符合：

* 国金证券 PTrade
* 10万元资金规模
* ETF CTA趋势跟踪策略
* 长期个人量化交易系统

的实际需求。
