## Phase 2: 核心指标层开发

- [ ] **Task 2.1** 实现移动平均线计算函数
  - **描述**：计算MA50、MA150、MA200、MA250等移动平均线
  - **参数配置**：从CONFIG读取窗口长度
  - **用途**：市场趋势判断、ETF个体趋势过滤
  - **验证**：计算结果与手动验证数据一致

- [ ] **Task 2.2** 实现动量模型计算函数
  - **描述**：实现多周期风险调整动量模型[reference:9]
  - **计算步骤**：
    1. 计算各周期收益率（250日、120日、60日、20日）
    2. 计算60日年化波动率进行风险调整
    3. 横截面Percentile Rank标准化（范围0~1）
    4. 加权合成动量得分（权重：50%、30%、15%、5%）
  - **验证**：单只ETF动量得分计算正确

- [ ] **Task 2.3** 实现趋势质量评分函数
  - **描述**：通过对数价格线性回归评估趋势质量[reference:10]
  - **计算步骤**：
    1. 对120日对数价格序列进行线性回归
    2. 提取斜率Slope（价格上涨速度）
    3. 提取R²（趋势连续性）
    4. 计算Quality Score = Standardized(Slope × R²)
  - **验证**：高斜率高R²资产得分更高

- [ ] **Task 2.4** 实现流动性评分函数
  - **描述**：基于60日日均成交额评估流动性[reference:11]
  - **计算步骤**：
    1. 计算每只ETF的ADV60日均成交额
    2. 横截面Percentile Rank标准化
  - **验证**：成交额越高的ETF得分越高

- [ ] **Task 2.5** 实现ATR函数
  - **描述**：基于20日最高最低价计算平均真实波幅[reference:12]
  - **计算**：ATR = MA(TR, window=20)，TR = max(H-L, |H-PC|, |L-PC|)
  - **用途**：仓位分配（1/ATR加权）、止损计算（3×ATR）
  - **验证**：ATR值在合理范围内

## Phase 4: 排名系统开发

- [ ] **Task 4.1** 实现最终评分计算函数
  - **描述**：综合动量、趋势质量、流动性三项得分计算最终排名[reference:18]
  - **加权公式**：Final Score = 0.70 × Momentum Score + 0.20 × Quality Score + 0.10 × Liquidity Score
  - **验证**：权重分配正确

- [ ] **Task 4.2** 实现ETF排序函数
  - **描述**：按Final Score从高到低对所有风险资产ETF排序[reference:19]

  - **验证**：排序结果可重现

- [ ] **Task 4.3** 实现选仓逻辑
  - **描述**：根据排名和附加条件确定持仓池[reference:20]
  - **入选条件**：
    - 市场评分 > 0
    - Final Score进入前5名
    - 收盘价 > MA200
  - **递补机制**：前5名中存在不符合条件的ETF时，依次递补第6名→第7名→第8名
  - **验证**：持仓数量恒定为5只

- [ ] **Task 4.4** 实现分散化约束检查
  - **描述**：限制同一行业和单一资产大类的持仓数量/权重[reference:21]

    - 单一资产大类：最大权重40%
  - **资产大类**：宽基、金融、科技、医药、消费、军工、资源
  - **验证**：持仓符合分散化要求


## Phase 5: 仓位分配与风控层开发

- [ ] **Task 5.1** 实现仓位分配函数
  - **描述**：基于风险平价（1/ATR）分配持仓权重[reference:22]
  - **计算步骤**：
    1. Raw Weight = 1 ÷ ATR20
    2. 标准化：Weight = Raw Weight ÷ Σ(Raw Weight)
    3. 应用约束：单只ETF ≤ 25%，单一资产大类 ≤ 40%
    4. 超出限制部分按比例重新分配
  - **验证**：权重合计=100%，约束边界正确

- [ ] **Task 5.2** 实现市场评分仓位调整
  - **描述**：将组合权重乘以市场仓位系数[reference:23]
  - **公式**：最终仓位 = 组合权重 × 市场仓位系数
  - **剩余资金**：配置现金ETF（511880）或现金
  - **验证**：不同市场评分下总仓位正确

- [ ] **Task 5.3** 实现退出规则检查函数
  - **描述**：实现所有退出条件判断[reference:24]
  - **退出条件**：
    - 动量退出：跌出排名前10名
    - 趋势退出：收盘价跌破MA200
    - ATR止损：收盘价跌破Entry Price − 3 × ATR20
    - 市场退出：市场评分降至0
    - 流动性退出：连续20个交易日成交额低于最低标准
  - **验证**：每个条件独立触发卖出信号

- [ ] **Task 5.4** 实现最大回撤控制
  - **描述**：当组合净值回撤超过20%时进入防御模式[reference:25]
  - **防御模式规则**：
    - 风险资产仓位减半
    - 停止新增仓位
    - 仅允许替换持仓
    - 恢复条件：净值回升后退出防御模式
  - **验证**：回撤触发时正确执行

- [ ] **Task 5.5** 实现状态管理模块
  - **描述**：所有策略状态统一保存在context.strategy_state[reference:26]
  - **状态结构**：

        "symbol": {

  验证：状态持久化正确
